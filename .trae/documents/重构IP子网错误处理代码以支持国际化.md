## 重构IP子网错误处理代码以支持国际化

### 问题分析
当前 `ip_subnet_calculator.py` 文件的 45-145 行定义了 `error_patterns` 列表，包含了硬编码的英文和中文错误信息模板。虽然代码已经导入了 `i18n` 模块的 `_` 函数，但没有充分利用现有的翻译系统。

### 解决方案
1. **移除硬编码的错误信息模板**：删除 `error_patterns` 列表中硬编码的英文和中文模板
2. **使用翻译键替换模板**：为每种错误类型定义唯一的翻译键
3. **修改错误处理逻辑**：在 `handle_ip_subnet_error` 函数中使用翻译系统获取错误信息
4. **添加翻译键到字典**：确保 `translations.json` 文件中包含所有新的翻译键

### 实施步骤
1. **分析现有错误模式**：梳理 `error_patterns` 列表中的所有错误类型
2. **定义翻译键**：为每种错误类型创建唯一的翻译键
3. **修改 `handle_ip_subnet_error` 函数**：
   - 移除 `error_patterns` 列表中的模板参数
   - 使用 `_` 函数获取翻译后的错误信息
   - 保持错误匹配逻辑不变，但使用翻译键替换硬编码文本
4. **添加翻译键到 `translations.json`**：为所有新的翻译键添加多语言支持
5. **测试功能**：确保所有错误信息都能正确翻译

### 预期效果
- 所有错误信息都通过翻译系统生成，支持多种语言
- 移除了硬编码的语言特定文本，提高了代码的可维护性
- 与现有国际化架构保持一致
- 支持动态语言切换

### 影响范围
- `ip_subnet_calculator.py` 文件的 `handle_ip_subnet_error` 函数
- `translations.json` 文件，需要添加新的翻译键

### 风险评估
- 低风险：修改仅涉及错误信息处理，不影响核心功能
- 确保所有翻译键都正确添加到 `translations.json` 文件
- 测试所有可能的错误情况，确保翻译正确
